# 「籤到」視覺總檢討與 v2 改造方案

> 2026-07-15 · Round 2。逐幕檢討 v1 截圖後的全面建議；標 ✅ 者本輪實作，標 ▶ 者列為後續。

## 總評

v1 的「深夜的廟」氛圍成立：暗場襯米黃籤紙的明暗架構、朱紅節制地只給筊杯／印章／按鈕，
方向正確。主要問題是**籤紙資訊密度遠低於真籤**（真籤詩紙是一張擠滿註解的資訊設計傑作，
v1 只有詩＋題＋籤等，顯得像文青明信片而非籤）、**裝飾語彙太「乾淨」**（真廟的美學是繁複
的：回紋、雲紋、對聯、匾額），以及**場景之間缺少空間連續感**（每幕像獨立卡片，不像在
同一座殿裡移動）。

## 分項檢討與建議

### 1. 籤紙（本輪重點 ✅）

| 問題 | 建議 |
|---|---|
| 資訊量太少，不像籤 | ✅ 補齊傳統版面元素：右側直排「籤到殿靈籤・第◯◯籤」、左側直排卦象＋五行、頂部吉凶改**匾額式大字**、詩下加**解曰四句**（四字格直排小欄，真籤紙的標配）、底部加**勸世語**一行 |
| 邊框只是兩條線 | ✅ 外框改**回紋（雷紋）連續邊框**（SVG pattern border-image），內框細線保留 |
| 籤等 chip 太現代 | ✅ 改為朱印風格方章（大吉／中平…直接蓋在紙右上），凶籤用靛墨章 |
| 紙面太新 | ✅ 加陳年紙斑（多層 radial-gradient）與紙纖維紋 |
| 卦象缺席 | ✅ 由 id 決定六十四卦（(id-1)>>1）＋陰陽變，顯示卦符（☰ U+4DC0 系列）＋卦名，deterministic 免資料欄位 |

### 2. 內容（本輪重點 ✅，另見 style-guide v2）

- v1 籤題把場景寫死（「報帳單被退」），投射空間反而變小——真籤詩用典故當鏡子。
  ✅ 全面改寫為七大意象體系：花木／本草／鳥獸／山川／時事／東史／西史（歷史題恰 7 字，
  「凱撒躊躇盧比孔」用傳統卦頭格式寫西方故事是本系統的招牌反差）
- ✅ 詩文禁用職場詞彙（時事批例外），幽默下沉到卦頭故事的觀察角度

### 3. 入殿幕

- 標題孤立、缺「殿」的感覺 → ✅ 「籤到」加**匾額框**（雙線框＋角釘），下緣紅絛
- ▶ 對聯（左右直排短聯）：手機寬度緊，暫緩
- 燈籠已修成扁圓＋直紋，✅ 再加燈穗

### 4. 擲筊幕

- 筊杯懸空無地面，落地無重量 → ✅ 加地面橢圓陰影，落地瞬間陰影收縮
- ▶ 筊杯 3D 翻轉用 WebGL 重做：現階段 CSS 動畫已達意，暫緩

### 5. 籤筒幕

- ✅ 筒身加回紋帶與「籤到殿」小字，強化道具感
- 搖晃進度條太像遊戲 UI → ✅ 改為筒下方「香灰色」細條，弱化存在感

### 6. 解籤區

- v1 已達標；✅ 微調：解曰改雙欄緊湊、被問類別放大置頂維持
- ▶ 解籤人語音（TTS 唸詩）：彩蛋候選，暫緩

### 7. 動效

- ✅ 展籤改「從籤筒抽出」的縱向拉出感（paper 進場位移加大、加旋轉回正）
- ✅ 蓋印動畫加落印震動（scale bounce 已有，加 rotate 抖動）
- `prefers-reduced-motion` 全部保留

### 8. 分享與導流（✅）

- 籤卡 PNG 底部加**站台網址**（依部署域名動態）；share sheet 帶每籤專屬頁連結；
  新增「複製連結」；每籤有靜態頁可被直接分享與收錄（見下）

### 9. SEO / AEO（✅）

- SPA 對爬蟲不友善 → 產 **128 張靜態籤詩頁**（`q/1.html`…`q/128.html`，含全文、
  canonical、per-籤 title/description/OG、CreativeWork JSON-LD、回首頁 CTA）
- **籤庫索引頁** `all.html`：128 籤總覽＋「什麼是籤到／怎麼求籤」FAQ（FAQPage JSON-LD，
  可見問答，AEO 主入口）
- `sitemap.xml`、`robots.txt`、`llms.txt`（給 AI 引擎的站台說明）
- index.html 補 noscript 內容、WebSite/WebApplication JSON-LD、完整 meta
- 產頁器：`tools/build_pages.py`（改 qian.json 後重跑，產物入版控，無 build 依賴）

## 色彩與字體（維持，微調）

- palette 不動（玄墨/朱紅/舊金/籤紙米黃/紙墨）；籤紙上朱紅使用量**增加**
  （回紋框、解曰標、勸世語框）以貼近「紅印籤紙」的傳統印刷感
- 字體維持文楷 TC＋Noto Serif TC；卦符用系統字型（U+4DC0–4DFF）
